Part 2¶

Write the LP

Since this is a network transmission problem, we will be defining the decision variables as the conversions between currencies, not the currencies themselves. There will one for each direction of the conversions (ie USD2EUR and EUR2USD)

I'm going to set this up as an array and use lpsolve. There are 8 currency types, so there will be 8 constraint equations and 56 [8*8 - 8] decision variables, assuming we're skipping conversions to the same currency.

This means that the constraint matrix will be fairly sparse, since JPY2CHF isn't going to affect the USD constraint equation, for example.

We will create the objective function at the same time, since we will be iterating through all combinations of the currencies anyways.

In [ ]:
# I could hard code this, but whatever. Feels like best practice to allow for flexibility.
num_currencies = len(currencies)

# initialize the constraint matrix
Aij = np.zeros((num_currencies,num_currencies**2))


# make an array of conversion rates, without the self-conversion values
conv_array = np.stack(conv_pd.values) - np.eye(num_currencies) 
# remember that we have to reduce by the transaction costs, which is the same
# as multiplying the conversion rate by 1-cost
eff_conv_array = conv_array*(1-np.stack(cost_pd.values))




# create the constraint matrix
# Since we're counting the amount moved to/from a currency IN THAT CURRENCY, the from values
# will just be multiplied by -1 whereas the to values will be with the transmision values.
for from_ind in np.arange(0,num_currencies):
    temp_array = np.zeros([num_currencies,num_currencies])
#     temp_array[from_ind,:] = -conv_array[from_ind,:] # everything we get rid of is negative
    temp_array[from_ind,:] = -1
    temp_array[:,from_ind] = eff_conv_array[:,from_ind] # everything we recieve is multiplied by the conversion rate, minus loss
    Aij[from_ind,:] = temp_array.reshape(num_currencies**2,)
Aij = Aij.tolist() # convert to a list


# create the objective function -- a flattened version of the transaction costs
obj = np.stack(cost_pd.values).reshape(num_currencies**2,) 
obj = obj.tolist() # convert to a list


# The minimum value we're willing to accept in USD is the amount we would get if we converted from
# JPY, EUR and CHF directly. So, we set the USD as a ge. Everything else needs to work out to what
# we had previously in that currency
#
#  16MJPY * conversion rate * (1-conversion cost)
USDmin = 16 * conv_pd.loc['JPY','USD']*(1-cost_pd.loc['JPY','USD']) + \
         6  * conv_pd.loc['EUR','USD']*(1-cost_pd.loc['EUR','USD']) + \
         30 * conv_pd.loc['CHF','USD']*(1-cost_pd.loc['CHF','USD'])

#      ['USD', 'AUD', 'GBP', 'EUR', 'INR', 'JPY', 'SGD', 'CHF']
rhs  = [USDmin,    0,     0,    -6,     0,   -16,     0,   -30]
comp = [     1,    0,     0,     0,     0,     0,     0,     0]

lp_handle = lp_maker(obj, Aij, rhs, comp, setminim=True)

Part 3¶

Put together the dual of the problem

We're fortunate that it's actually really easy to put together the dual of the problem. From Hillier and Lieberman, we just do the following transformations:

  1. Switch the problem from being a minimization to maximization problem
  2. Take the transpose of the LHS constraint array; this is the dual constraint array
  3. Swap the RHS of the constraints and the parameters of the objective function.

Programatically:
